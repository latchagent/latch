---
title: Quickstart (Docker)
description: Run Latch locally with Docker Compose and wrap an MCP server.
---

## 1) Start Latch

```bash
git clone https://github.com/latchhq/latch
cd latch
docker compose up
```

Open the dashboard at `http://localhost:3000`.

## 2) Create a workspace + upstream + agent

1. Register an account in the dashboard.
2. Create an upstream (for stdio MCP servers the name is just a label).
3. Create an agent and copy the **agent key**.

You’ll need these for the CLI:

- Workspace ID
- Upstream ID
- Agent key

## 3) Install the CLI

```bash
npm install -g @latchagent/cli
```

## 4) Configure the CLI

```bash
latch init
```

This writes a config file to `~/.latch/config.json`.

## 5) Wrap an MCP server

Example: GitHub MCP server:

```bash
latch run --upstream-command "npx" --upstream-args "-y,@modelcontextprotocol/server-github"
```

If an action requires approval, you’ll see an `APPROVAL_REQUIRED` error. Approve it in the dashboard and then retry with the provided approval token (your agent/tooling typically handles the retry).
